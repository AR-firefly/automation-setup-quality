# 使用指南

---

## 适用场景

任何涉及以下操作的场景，都建议先过一遍这个三步流程：

- 设置定时任务 / 自动化推送
- 配置数据同步规则
- 创建数据抓取或分析的自动化流程
- 修改已有自动化的数据源或筛选条件
- 任何需要 LLM/AI Agent 自动读取外部数据并做出判断的任务

**核心判断标准**：如果你不确定数据源长什么样就写了 prompt，那你需要这个流程。

---

## 快速上手

### 如果你是 AI Agent 用户

将 `SKILL.md` 的内容加载为 skill 或系统提示词：

- **Claude Code**：配置 skill 指向本目录，当触发词（设置自动化、创建自动化、配置 schedule 等）出现时自动加载
- **其他 Agent 平台**：复制 `SKILL.md` 全部内容作为系统提示词的一部分

### 如果你使用普通 LLM（ChatGPT / DeepSeek / Claude Web 等）

复制 `SKILL.md` 中"## 流程"之后的内容作为系统提示词。手动引导 LLM 走完三步：

> "请按自动化设置质量保证流程处理：先验证数据源，再出方案，写完后验证。"

### 如果你只想要方法论（不依赖任何 AI 工具）

直接阅读 `SKILL.md` 获取三步流程，`references/data-source-guide.md` 获取数据源验证方法。自己当自己的 QA。

## 场景演练

### 场景 1：首次设置一个新数据源

**背景**：要把某个知识库的文档定期摘要推送到群里。

**正确流程**：

```
Step 0：首次设置，数据源从未验证过 → 完整三步

Step 1：读数据源原始结构
  → 执行读取命令获取文档大纲
  → 确认有实质标题结构
  → 确认内容没有截断
  → 贴结果给对方："这是你要的文档吗？"
  → 对方确认：是

Step 2：出方案
  → 数据来源：知识库 A（token: xxx）
  → 筛选条件：本周新增的文档（创建时间 > 2026-07-04）
  → 评估标准：每篇文档摘要不超过 200 字
  → 输出格式：Markdown 列表推送到群
  → 对方确认：行，干

Step 3：写 + 验证
  → 写自动化
  → 手动跑一轮
  → 验证：数据源可读 ✅ 筛选正确（3 篇文档）✅ 摘要可读 ✅ 格式符合 ✅
  → 交付
```

### 场景 2：复用已验证的数据源

**背景**：场景 1 已经跑了一个月，现在要加一个新任务，读同一个知识库的另一个文件夹。

```
Step 0：数据源标识一致（同知识库，同 token）→ 跳过 Step 1

Step 2：直接出方案
  → 数据来源：知识库 A（token: xxx，已验证）
  → 筛选条件：不同文件夹，不同条件
  → 对方确认 → 写 → 验证
```

省了 Step 1 的数据源验证，但 Step 2 和 Step 3 不能省。

### 场景 3：对方说"你先动"

**背景**：对方说"别废话，直接干"。

```
Step 0：对方说"你先动" → 跳过 Step 2

Step 1：验证数据源（这个不能省）
  → 读数据源，贴结果确认
  → 对方确认：是

→ 直接写

Step 3：验证交付
  → 写完后验证，全部通过再交付
```

Step 1 和 Step 3 不能省。Step 1 是方向正确，Step 3 是结果正确。对方可以省讨论，但不能省验证。

## 自定义你的验证表

如果你的团队有特定平台（飞书、钉钉、企业微信、Notion 等），建议基于 `references/data-source-guide.md` 附录的空白验证表，创建一份你自己的版本：

1. 复制附录中的空白表格
2. 填入你们平台的具体命令和预期值
3. 遇到新的数据源类型就补充一行

> 建议团队共用一份验证表，新人上手直接查。

## 常见问题

### Q：什么时候可以跳过验证？

只在满足以下条件时可以跳 Step 1：

- 同一个数据源（相同 `<类型>:<标识>`）已经在当前任务中验证通过过
- 或者对方明确说"和上次一样"

**绝对不要因为"我之前看过"而跳过**——如果不确定就重验一次。

### Q：验证 3 轮没通过怎么办？

停止。不交付。告诉对方原因，让对方决定：

- 是不是数据源不对？
- 是不是筛选条件错了？
- 是不是预期不合理？

不要硬过 3 轮然后假装通过。3 轮不通说明你的假设偏差很大，需要人来介入。

### Q：数据源结构变了怎么办？

重新走 Step 1。数据结构变更不在"复用判断"的覆盖范围内。如果你怀疑结构可能变了，就跑一次 Step 1 确认。

### Q：这个流程适用于团队协作吗？

适用。建议：

- 团队共用一份验证表（`references/data-source-guide.md` 附录模板）
- 自动化设置完成后，验证结果记录在任务管理系统中
- 新人严格按照三步流程操作，熟练后可酌情跳过
